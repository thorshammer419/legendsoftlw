# Domain Glossary — The Legends of TLW

This file is the authoritative source for domain terms used in code and architecture discussions.
When a new concept is named in code or an architecture review, add it here.

---

## RoundContext

A dict that accumulates all inputs and outputs as the round pipeline progresses through
`resolve_round`. It is created after all blocking steps complete and passed to every
non-blocking step function (`_generate_action_lists`, `_generate_scene_image`,
`_send_notifications`, `_track_inactivity`).

**Shape:**

```python
{
    "campaign_id": str,
    "round_number": int,
    "campaign": dict,           # campaign document
    "story_state": dict,        # story state at the START of this round
    "campaign_players": list,   # all campaign_player docs
    "active_emails": list[str],
    "inactive_emails": list[str],
    "submitted_actions": dict,  # email → action data from pending_actions
    "characters": list,         # character docs at start of round
    "narrative": str,           # generated DM narrative text
    "state_update": dict,       # structured state extracted from narrative
    "new_story_state": dict,    # story state after apply_state_update (mutated by steps)
    "new_characters": dict,     # character docs after apply_state_update
    "scene_image_url": str|None,  # set by _generate_scene_image on success
}
```

**Invariant:** `scene_image_url` starts as `None`; `_generate_scene_image` mutates it in place
so the broadcast step picks up the URL without an explicit return value.

---

## Round

One complete iteration of the consensus-action cycle:

1. All active players submit actions (or the deadline fires).
2. `resolve_round` runs the pipeline.
3. A new DM narrative is broadcast to all players.
4. `round_status` returns to `"waiting"` and a new deadline is set.

Round 0 is the campaign intro — it fires once when the admin launches the campaign, is
generated by `campaign_intro_from_queue`, and is stored in the narrative log as `round: 0`.

---

## Round Status

Values of `story_state.round_status`:

- `"waiting"` — normal state; players may submit actions.
- `"resolving"` — pipeline is running; the idempotency check in `resolve_round` uses this
  to detect queue retries and avoid double-incrementing `round_number`.
